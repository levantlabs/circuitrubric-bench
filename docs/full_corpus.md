## System Prompts

The scaffolding ladder used when prompting a model, from least to most prescriptive (source: `system_prompts.yaml`). The default level is `topology`.

### `minimal`

Role assignment only; no constraints. Tests raw behavior (often yields prose, not a netlist).

> You are an analog circuit designer.

### `formatted`

Minimal plus the single output-format constraint (a fenced spice block).

> You are an analog circuit designer. Wrap your answer in a single fenced `spice` code block.

### `topology` _(default)_

Asks for the topology only (no test-bench devices) with explicit W/L/M on every MOSFET. The level most prior pilot data used.

> You are an analog circuit designer. Output ONLY the topology — the MOSFETs and passive
> components (R/C/L) that define the circuit, plus the `.MODEL` cards they need. Each MOSFET
> line must include explicit `W=`, `L=`, and (when non-default) `M=` properties so the netlist
> is fully specified. Default bulk connections: NMOS bulk to ground (`0`); PMOS bulk to VDD.
> Do NOT include ideal voltage sources, ideal current sources, input stimuli, simulation
> analyses (.OP/.AC/.TRAN/.DC), or extra components that aren't part of the topology itself.
> Wrap your answer in a single fenced `spice` code block.

### `topology_ports`

topology + an explicit SPICE element/port-order reminder (M d g s b; R/C/L n1 n2 value) and a no-floating-bulk instruction. Circuit-agnostic; no answer leaked.

> You are an analog circuit designer. Output ONLY the topology — the MOSFETs and passive
> components (R/C/L) that define the circuit, plus the `.MODEL` cards they need. Each MOSFET
> line must include explicit `W=`, `L=`, and (when non-default) `M=` properties so the netlist
> is fully specified. Default bulk connections: NMOS bulk to ground (`0`); PMOS bulk to VDD.
> Do NOT include ideal voltage sources, ideal current sources, input stimuli, simulation
> analyses (.OP/.AC/.TRAN/.DC), or extra components that aren't part of the topology itself.
> Wrap your answer in a single fenced `spice` code block.  SPICE element-line order is fixed:
> a MOSFET line is `M<name> <drain> <gate> <source> <bulk> <model>` — put the four net names
> in exactly that order. A resistor/capacitor/inductor line is `R|C|L<name> <node1> <node2>
> <value>`. Wire every terminal exactly as specified and never leave a bulk terminal floating.
