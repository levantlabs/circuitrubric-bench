## Architecture at a glance

```

  SPICE text  тФАтФАтЦ║ Netlist  тФАтФАтЦ║  typed bipartite graph  тФАтФАтЦ║  grade()  тФАтФАтЦ║ GradeResult (Credit)

                   nets)            device nodes,
                                    terminal edges)

  fixtures/NNN_*/  тФАтФАload_taskтФАтФАтЦ║  Task (prompt, references, ratio_groups, meta)

  run: Task тФАpromptтЖТ Backend(LLM) тФАrawтЖТ extract_netlist тФАтЦ║ grade() тФАтЦ║ summary.jsonl row
```

Two entry points:
- **Grade** an existing netlist: `grade(text, task)` тЖТ `GradeResult`. Network-free and
  side-effect-free; reads only the fixture's reference `.cir` files.
- **Run**: generate netlists with an LLM and grade them (`runner` + `backends`), writing a
  per-run directory.

The **grader core** (`netlist`, `task`, `graph`, `ratio`, `grader`) has no LLM dependencies.
The **reference runner** (`runner`, `backends`, `cli`) sits on top; LLM SDKs are optional extras.

### `netlist.py` тАФ parsing
`parse_netlist(text) -> Netlist`. Line-oriented SPICE parse:
- `M<name> <drain> <gate> <source> <bulk> <model> [W= L= M= тАж]` тЖТ MOSFET `Device`.
- `R`/`C`/`L`/`V`/`I` тЖТ two-terminal `Device` with `n1`/`n2` and a `value` (handles `V1 a 0 DC 5`).
- `.MODEL <name> <NMOS|PMOS> (...)` cards resolve a device's model name тЖТ type in a second pass.
- **Net names are lowercased** at parse time (SPICE is case-insensitive; fixes a class of false NONEs).
- If a MOSFET's type was never declared by a `.MODEL` card but the model *name* encodes polarity
  (`PMOS_MODEL`, `nmos_dev`), the type is inferred **and** `Netlist.has_inferred_types=True` тАФ which
  the grader uses to cap credit at `TOPOLOGY` (no `.MODEL` тЗТ not a complete/FULL answer).

Data: `Device(name, type, terminals: dict, properties: dict)`, `Netlist(devices, nets, has_inferred_types)`.

### `task.py` тАФ fixtures (pydantic)
`load_task(dir) -> list[Task]`, one `Task` per prompt variant (they share references + ratio_groups).
- `RatioGroup` validates lengths and MOSFET-vs-passive consistency at load.
- `Task(id, topology_id, prompt_id, prompt, references: list[Path], ratio_groups, meta)`.
- `references = sorted(dir.glob("reference*.cir"))` тАФ a submission passes if it matches **any**.
- `list_tasks(root)` flattens all fixture dirs (used by tests).

### `graph.py` тАФ the isomorphism core
`build_graph(netlist)` тЖТ a `networkx.MultiGraph`: one node per net (`kind="net"`), one per device
(`kind="device"`, `type=`), one edge per terminal (`terminal=` role label).
- **Matching** (`isomorphic`, VF2 via `MultiGraphMatcher`): node `kind`+device `type` must match;
  edges match on the sorted multiset of terminal labels.
- **Terminal symmetry** (`_TERMINAL_CANONICAL`): passive `n1`/`n2` collapse to one label (a flipped
  resistor still matches). **MOSFET `drain`/`source` are kept distinct** тАФ a swap is a real error
  for the credit ladder.
- Two **canonical-graph** variants relax one convention each, for diagnostics:
  - `build_bulk_canonical_graph` тАФ redirects every MOSFET bulk to a sentinel net тЖТ "iso modulo bulk"
    (drives `PARTIAL_BULK`).
  - `build_sd_canonical_graph` тАФ collapses `drain`/`source` to one label тЖТ "iso modulo S/D
    orientation" (drives the `functional` diagnostic).
- `subgraph_isomorphic` тАФ reference as a node-induced subgraph of a larger submission (drives
  `DECORATED`). `iso_device_mappings` yields refтЖТsub device-name maps for the ratio check.

### `ratio.py` тАФ relative sizing
`check_ratio_groups(netlist, groups) -> RatioCheckResult`. Per group:
- MOSFET groups check **effective W = W ├Ч M**, plus L (and M separately if `ratio_M` declared),
  within **┬▒1%**. Passive groups check `value`.
- `all_ok` тЗТ everything passes; the grader maps a failing-M-only result to `PARTIAL`.

### `grader.py` тАФ putting it together
`grade(text, task) -> GradeResult`. Flow: parse тЖТ for each reference, try `iso_device_mappings`;
under each mapping run the ratio check; keep the best `Credit`. If no strict iso: try
bulk-canonical (тЖТ `PARTIAL_BULK` when every violation is bulk-tied-to-source), else subgraph
(тЖТ `DECORATED`), else `NONE`. `sd_equiv`/`functional_full` are computed alongside (never affect
`credit`).

`Credit` ladder (rank highтЖТlow): `FULL > PARTIAL > PARTIAL_BULK > TOPOLOGY > DECORATED > NONE`.
`_credit_from_rc`: ratios all-ok тЖТ `FULL`; only M-ratio fails тЖТ `PARTIAL`; any W/L/value fails тЖТ
`TOPOLOGY`. `has_inferred_types` caps any iso match at `TOPOLOGY`.

`GradeResult` fields: `credit`, `parse_ok`, `iso_ok`, the four ratio flags, `matched_reference`,
`error`, `topology_recognized`, `extra_devices`, and diagnostics `sd_equiv` / `functional_full`.

### `runner.py` тАФ generate + grade
- `RunConfig` тАФ backend, model, tiers, reps, max_tokens, system prompt, reasoning settings, etc.
- `extract_netlist(raw)` тАФ pull the **last** fenced ```` ```spice ```` block; fall back to a bare
  fence that looks like SPICE; final fallback accepts an unfenced body iff the real parser finds тЙе2
  devices (prose can't slip through).
- `run_one` тАФ **resume-safe**: reuse a non-empty saved raw; atomic writes (tmp + `os.replace`);
  retry transient API errors (up to 3 attempts); one bad response never aborts the run.
- `run_benchmark` тАФ builds the `product(topologies, prompts, reps)` plan, writes `config.json`
  (with `git_rev`) + `summary.jsonl`; aborts a resume on config drift (model/backend/prompt change).
- `run_sweep` / `format_combined_table` тАФ multi-model sweep тЖТ a leaderboard table.
- `_grade_to_dict` тАФ serializes `GradeResult` into each summary row (the schema downstream tools read).

### `backends.py` тАФ LLM backends
A `Backend` Protocol with one `call(model, temperature, max_tokens, system_prompt, user_prompt)
-> dict` (normalized: `raw_text`, `stop_reason`, `latency_ms`, `usage`). Three impls:
- `AnthropicBackend` тАФ Messages API.
- `OpenAICompatibleBackend` тАФ `/v1/chat/completions`; covers OpenAI, vLLM, Together, OpenRouter,
  ollama's `/v1`, тАж via `base_url` + `api_key_env`; supports an OpenRouter `reasoning` object.
- `OllamaNativeBackend` тАФ native `/api/chat` with `think` control (so reasoning models don't burn
  the budget in a hidden trace); discards the thinking trace, grades only visible content.

SDKs are **lazy-imported** with a clear install hint; API keys come **only** from env vars
(`ANTHROPIC_API_KEY`, or a configurable `api_key_env`). `make_backend(id, тАж)` constructs by id.

### `cli.py`
Argparse front end. `show` (inspect a fixture), `grade` (one netlist; exit code encodes the credit),
`grade-all` (a directory of `<fixture_id>.cir`), `run` (one model end-to-end), `run-all` (sweep from
`models.yaml`). System prompt via `--system-prompt-id` (looked up in `system_prompts.yaml`) or a
literal `--system-prompt`.

## Extending

- **Add a backend** тАФ implement the `Backend` Protocol in `backends.py` and register it in
  `make_backend`; keep the normalized return dict.
- **Add a system prompt** тАФ append an entry to `system_prompts.yaml` (`id` + `text`); usable
  immediately via `--system-prompt-id`.
- **Add a fixture** тАФ create `fixtures/NNN_<slug>/` with the four files, then run
  `scripts/validate_fixtures.py` (it enforces the conventions and that the reference grades FULL).
- **Add an accepted alternative topology** тАФ drop a `reference_alt_N.cir` in the fixture dir; a
  submission passes on any blessed reference.
